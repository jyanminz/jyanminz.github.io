迭代器与traits编程技术

一、迭代器

  1. STL的中心思想在于：将容器与算法分开，彼此独立设计最后再使用一种技术将二者结合在一起。这种技术就是迭代器。

  2. 迭代器是一种智能指针，其最重要的编程工作就是对于operator*和operator→的重载工作。

  3. 每一种STL容器都有提供专属迭代器；

  4. 在算法中使用迭代器时，很有可能会使用到其相应型别，最常用的相应型别有五种：value_type、difference_type、pointer、reference和iterator_category，一般的型别可以通过模板的参数类型推导机制来取得，但有些型别不能，因此我们需要使用到traits编程技术。

  5. 为了符合规范，任何迭代器都应该提供五个内嵌相应型别，以利于traits来萃取；为了方便，STL提供了一个iterator class，如果我们令新设计的迭代器都继承自他，就可以保证符合STL的规范

最后着重介绍一下迭代器的第五种型别：iterator_category；根据移动特性与实施的操作，可以将迭代器分为五种类型：

  1. input iterator：该种迭代器所指的对象，不允许外界改变，是read-only
  2. output iterator：只写write-only
  3. forward iterator：允许写入型算法在该种迭代器所形成的区间上进行读写操作
  4. bidirectional iterator：可以双向移动
  5. random access iterator：涵盖所有指针的算数能力

三、SGI STL的私房菜：__type_traits

上述提到的iterator_traits负责萃取迭代器的特性，__type_traits负责萃取型别的特性，这里关注的型别特性是指：

  1. 该型别是否具备non-trivial default ctor ？
  2. 是否具备non-trivial copy ctor ？
  3. 是否具备non-trivial assignment operator ？
  4. 是否具备non-trivial dtor ？

通过上述信息，我们可以在对该型别进行如构造、析构、拷贝、赋值等操作时采取最有效率的措施；比如说不调用身居高位、不谋实事的那些ctor、dtor，而采用内存直接处理操作如malloc、memcpy等来获得最高的效率。
